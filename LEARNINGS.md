# Learnings

### 2026-03-06 (Audit fixes — cross-cutting patterns)
- [res_ok_guard]: Always check `if (!res.ok) throw new Error(...)` before `res.json()`. Without this, HTML error pages (e.g. 502 gateway) cause `SyntaxError: Unexpected token '<'` which is cryptic to users. Apply to every fetch→json path.
- [encoding_detection]: For text file extraction, UTF-8 is not always correct. Pattern: check for UTF-8 BOM (EF BB BF), try UTF-8, check for replacement character `\uFFFD` — if present, re-decode as Windows-1252. Works for server (`Buffer.toString`) and browser (`TextDecoder`).
- [page_local_concurrency]: Don't duplicate global settings as page-local state. `systemSettings.maxConcurrency` from the store is the single source of truth for `pLimit()`. Removing per-page ± buttons reduces confusion.
- [inline_component_focus_bug]: React components defined inside another component (as arrow functions) get re-created on every render, causing input focus loss. Fix: move them to top-level functions and pass data via props. Applied to `WorkerCard` in consensus-coder.
- [auto_accept_ux]: Auto-accepting AI suggestions without user action is confusing — users don't know what changed. Better pattern: show suggestions visually distinct (lighter background), require explicit "Accept All AI" click. Added "Dismiss" button to clear suggestions.
- [manual_coder_deleted]: Manual Coder removed entirely. AI Coder with enhanced suggestion UX covers the same use case. Must remove from sidebar, home page, AND any other nav references.
- [datatable_column_filters]: Per-column filter inputs below headers + multi-format export (CSV/TSV/JSON) are cross-cutting enhancements — changing `DataTable.tsx` once improves every module that uses it.

### 2026-03-02 (Provider store defaults — Ollama conflict fix)
- [ollama_default_enabled]: `ollama` was `isEnabled: true` in `DEFAULT_PROVIDERS` in `store.ts`. This caused Ollama to auto-activate on every fresh install and conflict with LM Studio (both listen on adjacent local ports). Fix: set `isEnabled: false` for all local providers (`ollama`, `lmstudio`, `custom`). Local providers must always be opt-in.
- [zustand_persist_merge]: The `merge` function in zustand `persist` merges saved state over `DEFAULT_PROVIDERS`. This means a `isEnabled` default change in code will NOT affect existing users who already have the key saved in localStorage. Only new users see the corrected default. Existing users must toggle manually in Settings.
- [ollama_default_model]: Changed `ollama.defaultModel` from the non-standard `gpt-oss:latest` to `llama3` (the canonical Ollama default).
- [local_provider_pattern]: All local providers (`isLocal: true`) use a dummy `apiKey` string (`"ollama"`, `"lm-studio"`) rather than a real credential. Zod validation uses `apiKey: z.string().default("")` (not `.min(1)`) to allow these.

### 2026-03-02 (Process Documents enterprise upgrade)
- [pdf_parse_v2]: pdf-parse v2.x (installed as ^2.4.5) uses a class-based API, NOT the v1 function export. Use `new PDFParse({ data: buffer })` then `await parser.getText()` which returns `{ text, pages }`. The old `pdfParseModule.default ?? pdfParseModule` pattern only works for v1. PasswordException is exported as a named export.
- [pdf_image_only]: pdf-parse silently returns empty text for image-only PDFs. Must check `result.text.trim() === ""` after parsing and throw a user-friendly message. Same applies to pdfjs-dist in the browser.
- [pdf_password]: pdf-parse/pdfjs throws errors whose message contains "password" or "PasswordException" for encrypted PDFs. Classify these separately from corrupt/invalid PDF errors.
- [pdfjs_per_page_trycatch]: Wrap `getPage()` and `getTextContent()` in a per-page try/catch (skip failed pages rather than aborting). A single bad page should not fail the whole extraction.
- [extract_result_shape]: Returning `{text, truncated, charCount}` from extractors lets callers show truncation warnings and emit metadata. Much better UX than silent truncation.
- [field_def_schema]: `FieldDef {name, type, description}` is the pattern for typed extraction schemas. Passing the schema in the system prompt (replacing `{schema}` placeholder) significantly improves LLM extraction quality vs free-form column names.
- [document_analyze_3k]: For AI field suggestion, only the first 3000 chars of a document are needed. Keeps the analysis call fast and cheap while giving the LLM enough context to suggest a schema.
- [plimit_concurrent_state]: When using pLimit for concurrent file processing, use `Map<index, results>` to collect results and iterate in order after `Promise.allSettled`. Avoids race conditions on accumulated arrays.
- [preset_localStorage_hydration]: Field presets stored in localStorage must be loaded in `useEffect(() => {}, [])`, not in `useState` initializer — the initializer runs during SSR and causes hydration mismatches.
- [prompt_count_now_18]: prompts.test.ts now asserts 18 prompts (was 16). Update when adding new prompts.
- [format_extraction_schema]: `formatExtractionSchema(fields: FieldDef[]): string` in `prompts.ts` renders a FieldDef array as a numbered list with types/descriptions plus a `CSV header: col1,col2,...` line. This fills the `{schema}` placeholder in the `document.extraction` prompt. Much better LLM compliance than passing raw JSON.
- [csv_first_extraction]: For document extraction, instruct the LLM to output raw CSV only. Attempt `parseCsvResponse()` first; fall back to JSON parse if CSV yields 0 records; final fallback is `[{extracted_text: rawOutput}]`. This pattern handles models that ignore format instructions.
- [csv_parser_quoted_fields]: The CSV parser must handle RFC 4180 quoted fields: `""` inside quotes = escaped quote; `,` inside quotes = literal comma. Implement character-by-character with `inQuotes` toggle. The simple `split(",")` approach breaks on any realistic document data.
- [document_prompts_forbidden]: Adding an explicit STRICTLY FORBIDDEN section (markdown, code blocks, JSON, prose, "null"/"N/A") to extraction prompts significantly reduces LLM formatting noise. State what is forbidden, not just what is allowed.
- [field_editor_quick_add]: Quick-add input parsing `"author, date:date, price:number"` — split on comma, for each token split on `:` to get `[name, type?]`. Default type to `"text"` if absent or invalid. Skip empty tokens. Dedup by name before appending. Makes field entry keyboard-friendly.
- [csv_header_import]: To import field names from a CSV, read only the first line with FileReader (`reader.onload` → split on `\n` → take `[0]` → split on `,`). Map each header to `{name: h.trim(), type: "text", description: ""}`. Fast and zero-dependency.
- [ai_suggest_phase]: A two-phase Analyze→Extract workflow: Phase 1 sends only first 3000 chars to the LLM and asks for `FieldDef[]` as JSON. Phase 2 uses the confirmed fields to run full extraction. Phase 1 is cheap (<1K tokens) and gives users a starting point they can edit before committing to a batch run.

### 2026-02-25 (Abstract Screener module)
- [ris_parser]: RIS bibliographic format uses `TAG  - value` lines with `ER  -` as record separator. Parsing: split on `/^ER\s*-/m`, then for each record match `/^([A-Z][A-Z0-9]?)\s{1,3}-\s(.*)$/`. Multi-value tags (AU, KW) must be accumulated as arrays then joined with `"; "`. Skip records with neither title nor abstract.
- [react_highlight_split]: To highlight terms in React without dangerouslySetInnerHTML: use `text.split(/(pattern)/gi)` with capturing group, then map — odd-indexed parts are the captured matches. Avoids global lastIndex mutation from `re.test()` in a map closure.
- [ris_dropzone]: react-dropzone needs explicit MIME types for `.ris`. Use both `"application/x-research-info-systems": [".ris"]` and `"text/plain": [".ris"]` since browsers may treat it as plain text.
- [prompt_test_count]: The `prompts.test.ts` hardcodes a count of registered prompts. Update the test whenever new prompts are added — both the count assertion and the `expectedIds` array.
- [screener_config_panel]: For a multi-step tool (config → batch → review), use a `skipConfigPanel` boolean state rather than deriving visibility purely from AI result count — allows "Skip AI / manual review" use case without breaking the post-batch state.
- [tauri_build_static_pages]: Tauri static export (`npm run build:tauri`) produces 15 static pages including abstract-screener — no dynamic routes needed for the screener since all processing is client-side.

### 2026-02-22 (Phase B: Electron removal + Tauri no-sidecar migration)
- [tauri_phase_b_llm_browser]: Vercel AI SDK's `generateText()` is pure `fetch` with no Node.js APIs — runs directly in WKWebView/WebView2. All LLM calls can move to the browser, eliminating the Node.js sidecar entirely. Bundle drops from ~85 MB → ~10 MB; startup is instant.
- [tauri_static_export]: Next.js `output: 'export'` (static) is needed for Tauri Phase B (no server). Cannot coexist with API route handlers or dynamic routes without `generateStaticParams`. Solution: shell script in `build:tauri` that temporarily moves `src/app/api/` and `src/app/history/[id]/page.tsx` to `/tmp`, runs `TAURI_BUILD=1 next build`, then restores via `trap ... EXIT`.
- [nextjs_export_dynamic_routes]: `generateStaticParams() { return [] }` is NOT enough for `output: 'export'` — Next.js treats an empty array the same as "missing" and errors. The route must either provide actual pre-rendered params or be excluded from the build. Also: `generateStaticParams` cannot be in a `"use client"` file — must be in a server component.
- [tauri_spa_navigation]: In Tauri static export, client-side navigation (Next.js router `history.pushState`) does NOT trigger new file requests — the WebView doesn't reload. Dynamic routes that can't be pre-rendered can be handled inline in the parent page via search params (`?id=uuid`). The `[id]/page.tsx` is excluded from the Tauri build but the client component remains importable.
- [tauri_plugin_sql]: `@tauri-apps/plugin-sql` lets TypeScript execute SQL directly via `Database.load('sqlite:path.db')` → `.execute(sql, params)` / `.select(sql, params)`. Eliminates need for Prisma, custom Rust commands for DB operations. Migrations registered in `main.rs` via `SqlBuilder::add_migrations()`.
- [tauri_isTauri_hydration]: Module-level `const isTauri = typeof window !== 'undefined' && '__TAURI_INTERNALS__' in window` is fine for most pages. BUT when used to determine `<Link href>` in static export, it causes hydration mismatch (SSR=false, client=true). Fix: use `useState(false)` + `useEffect(() => setIsTauri(...), [])` when the value affects rendered output; or use `onClick` + `router.push` instead of `<Link>`.
- [tauri_capabilities]: All Tauri plugins must be listed in `src-tauri/capabilities/default.json`. Missing permission = silent failure or runtime crash. `sql:default`, `sql:allow-execute`, `sql:allow-select`, `sql:allow-load` are needed for tauri-plugin-sql.
- [nextjs_useSearchParams_suspense]: `useSearchParams()` from `next/navigation` in a `"use client"` component requires the component to be wrapped in `<Suspense>`. Otherwise Next.js throws during build. Pattern: split into `HistoryContent` (uses hook) + `HistoryPage` wraps with `<Suspense fallback={...}>`.

### 2026-02-22 (Tauri polish + export + Manual Coder UX)
- [tauri_webview_download]: WKWebView (macOS system WebView used by Tauri) does not support the HTML `download` attribute on anchor tags. Blob URL + `a.click()` silently does nothing. Fix: detect Tauri via `"__TAURI_INTERNALS__" in window` (Tauri v2; v1 used `"__TAURI__"`) and invoke a Tauri command that calls `tauri_plugin_dialog::blocking_save_file()`.
- [tauri_window_state]: `tauri-plugin-window-state` auto-restores window size/position on plugin init — no manual call needed. `AppHandleExt` only exposes `save_window_state`, not `restore_window_state`.
- [tauri_async_blocking_dialog]: `blocking_save_file()` from tauri-plugin-dialog can be called directly inside an `async fn` Tauri command. It dispatches to the main thread internally; no `spawn_blocking` needed on macOS.
- [tauri_db_path_production]: In Tauri production bundles, the CWD of the sidecar is unpredictable. Always pass `DATABASE_URL` explicitly via `.env("DATABASE_URL", path)` using `app.path().app_data_dir()` → `~/Library/Application Support/{bundle_id}/`.
- [shared_export_util]: Centralising CSV build + download in `src/lib/export.ts` as an async function enables Tauri/browser branching with zero impact on call sites (use `void downloadCSV(...)`). Dynamic import of `@tauri-apps/api/core` inside the Tauri branch avoids bundling it in the browser build.
- [tauri_detect_v2]: Use `"__TAURI_INTERNALS__" in window` to detect Tauri v2 at runtime. `"__TAURI__" in window` is Tauri v1. Both can coexist but v2 is the current standard.
- [settings_deny_list]: Claude Code `settings.json` deny list blocks `git add/commit/push` by default. Removing those entries allows Claude to commit directly; `git merge/rebase/reset` remain blocked for safety.

### 2026-02-22 (web stability + desktop packaging)
- [stability_db_log]: In Next.js API routes, DB log writes (Prisma) that follow a successful LLM call must be wrapped in their own try/catch. Without isolation, a DB failure bubbles to the outer catch and returns 500 to the client even though the LLM result is available — a silent data loss bug.
- [stability_workers]: Use `Promise.allSettled` (not `Promise.all`) for parallel LLM worker arrays. Define a minimum success threshold (≥2). Individual worker failures should produce a partial result, not abort the entire analysis.
- [stability_retry]: `withRetry` should classify non-retryable errors (401, 403, invalid_api_key, authentication) and throw immediately. Auth errors retrying 3× wastes ~3–9s and burns rate limit quota unnecessarily.
- [validation_local]: Zod schemas for LLM routes should use `apiKey: z.string().default("")` not `.min(1)`. Local providers (Ollama, LM Studio) pass an empty string or provider-specific token; enforcing min(1) silently breaks local-only workflows.
- [desktop_standalone]: `output: "standalone"` in `next.config.ts` produces `.next/standalone/server.js` — a self-contained Node.js server. This enables both Electron (uses built-in Node) and Tauri (sidecar) packaging without touching any app code. Has zero effect on web deployment.
- [desktop_electron]: Electron's built-in Node.js (via `process.execPath`) can spawn the standalone Next.js server. No separate Node.js installation needed. Port 3947 avoids clashing with dev server on 3000. Pattern: `spawn(process.execPath, ['server.js'])` → poll port → `loadURL`.
- [desktop_tauri_sidecar]: Tauri Phase A uses `tauri-plugin-shell` to spawn Node.js as a sidecar. Requires a platform-suffixed binary in `src-tauri/binaries/node-{target-triple}`. Bundle size ~85 MB (vs Electron ~160 MB). Phase B migration eliminates the sidecar by moving LLM calls to browser fetch + Prisma to tauri-plugin-sql.
- [hydration_localstorage]: Reading localStorage in `useState` lazy init (with `typeof window !== 'undefined'` guard) causes React hydration mismatches because SSR takes the else branch and renders different HTML. Fix: always init state with safe server default, read localStorage only in `useEffect(() => {}, [])`, guard persist effects with `isMounted` flag.

### 2026-02-16
- [prompt_registry]: System prompts are centrally managed in `core/prompt_registry.py` via `PromptRegistry` class. Each prompt has an ID, name, description, category, module, and default value. The Settings page groups them by category. New modules need explicit registration here to appear in the UI.
- [prompt_registry]: Transform Data, AI Coder, and Model Comparison previously had no registered system prompts. Transform and Model Comparison use user-entered prompts via `st.text_area`; AI Coder builds prompts dynamically via `_build_ai_prompt()`. All three now have defaults registered.
- [session_state]: AI Coder uses `aic_` prefix, Manual Coder uses `mc_` prefix for all session state keys. Changing a key name (e.g. `aic_text_col` to `aic_text_cols`) requires updating init, save, load, config, all rendering, and all processing methods.
- [backward_compat]: When changing session save format (e.g. `text_col` string to `text_cols` list), the load function must handle both old and new formats. Pattern: `text_cols = data.get("text_cols"); if text_cols is None: old_col = data.get("text_col"); text_cols = [old_col] if old_col else []`.
- [streamlit_multiselect]: `st.multiselect` with a `key` parameter stores its value in session state under that key. The `default` parameter only applies on first render; subsequent renders use the session state value. This means changing `default` after first render has no effect.
- [coder_architecture]: Both AI Coder and Manual Coder follow identical patterns: `render_config()` returns `ToolConfig` with `config_data` dict, `render_results()` reads from session state. The immersive dialog and regular view both need updating when changing data access patterns.
- [text_display]: When displaying multiple columns in coders, plain text format uses `[ColName]: value` (for AI prompts), HTML format uses `<b>ColName:</b> value` (for UI display). Single column returns raw value without labels for backward compatibility.
- [testing]: All 488 tests pass via `pytest tests/ -x -q`. Tests are in `tests/` with unit tests under `tests/unit/`. The test suite covers coding tools, settings, automator, codebook, consensus, error classifier, LLM client, model comparison, models, and providers.
- [project_docs]: Project markdown docs live in root (`README.md`, `CONTRIBUTING.md`, `AUTOMATOR_UX_CHANGES.md`, `OPENROUTER_INTEGRATION.md`) and `docs/` (`system-prompts.md`, `sample-data-and-prompts.md`, `CONSENSUS_ENHANCED_JUDGE.md`). When adding features, update `docs/system-prompts.md` and `docs/sample-data-and-prompts.md` for prompt-related changes.
- [run_app]: The app runs via `.venv/bin/streamlit run app.py`. The `run.sh` script doesn't activate the venv, so running `streamlit` directly from shell fails with "command not found". Always use the venv path.

### 2026-02-22 (autosave)
- [crash_recovery]: Dual-slot autosave pattern: write current to AUTOSAVE_KEY, rotate old AUTOSAVE_KEY → AUTOSAVE_PREV_KEY before overwriting. Restore checks primary then falls back to prev. Gives one level of undo against corrupt writes.
- [beforeunload_ref]: `beforeunload` handler must read state via a `useRef` (stateRef) not closures, because it's registered once (`[]` dep array) and can't close over fresh state. A render-sync `useEffect` with no dep array keeps stateRef current after every render.
- [recovery_ux]: Replace toast.info for autosave restore with persistent amber banner (`recovered` state). Banner shows session name, coded count, and relative time. Dismissed by user with ✗. Cleared by `doDataLoaded` when fresh data loads.
- [pending_load_guard]: When `codedCount > 0`, intercept `handleDataLoaded` → set `pendingLoad` state → show Dialog confirmation. Cancel keeps session; "Load anyway" calls `doDataLoaded`. Protects against accidental drag-drop destroying work.
- [autosave_indicator]: `autosaveTime` state + `timeAgo` ticker effect (15s interval) → "✓ Autosaved just now / 15s ago" in session bar. Shows only when `autosaveTime` is set (i.e. data loaded).
